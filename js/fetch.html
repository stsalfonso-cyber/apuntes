El método Fetch

El método fetch() es la forma moderna y estándar que utiliza JavaScript para hacer peticiones HTTP a servidores (APIs) y obtener datos (como listas de productos, precios o información de usuarios).

El flujo: fetch → response → .json() → data.

Funcionamiento básico:


fetch('https://api.spagnolo.es/productos')
     .then(response => {
         // Aquí recibes el objeto de respuesta
         console.log(response.status); // 200 en caso de éxito
         return response.json();        // Convertimos el cuerpo de la respuesta a JSON
     })
     .then(data => {
        // Aquí ya tienes los datos listos para usar
         console.log(data);
     })
     .catch(error => {
         // Manejo de errores de red (ej: no hay conexión o la URL no existe)
         console.error("Hubo un problema:", error);
     });
    Conceptos claves:
  • Petición asíncrona: El código no espera a que el servidor responda para seguir ejecutándose; por ello usamos promesas (.then()).
  • Objeto Response: fetch no devuelve directamente los datos, devuelve un objeto de respuesta que contiene información como el status (ej: 200 OK, 404 Not Found, 500 Server Error).
  • Parseo de datos: El cuerpo de la respuesta suele venir en formato stream. Para extraerlo, debemos invocar métodos como .json(), que a su vez devuelven otra promesa.
Validar la respuesta:

Un error común es pensar que un error 404 o 500 dispara el .catch(). No es así. El .catch() solo se dispara si la red falla (ej. estás sin internet).

Para manejar errores de servidor, debes verificar la propiedad .ok:


fetch('https://api.spagnolo.es/productos')
    .then(response => {
        if (!response.ok) { // Verificamos si la respuesta fue exitosa
            throw new Error("Error en la respuesta: " + response.status); // Si no es exitosa, lanzamos un error para que se maneje en el catch
        }
        return response.json();
    })
    .then(data => {
        console.log(data);
    })
    .catch(error => {
        console.error("Hubo un problema:", error);
    });

Resumen técnico:

Paso Acción ¿qúe es?
fetch(url) Inicia la petición HTTP Devuelve una Promesa (esperando la respuesta).
.then(response => ...) Recibe el objeto Response Este objeto contiene la información de la respuesta del servidor. Pero no los datos
response.json() Convierte el cuerpo de la respuesta a JSON Devuelve otra Promesa que se resuelve con los datos parseados en formato JSON. Hace lo mismo que JSON.parse()
.then(data => ...) Recibe los datos parseados Aquí ya tienes los datos listos para usar en tu aplicación.
