Async / Await

Async / Await es una sintaxis que permite escribir código asíncrono de manera más legible y sencilla. Facilita la manipulación de promesas y evita el uso excesivo de callbacks.

  • Declaración async:

    • Al añadir async delante de una función, esta garantiza que siempre retornará una promesa.

    
async function obtenerTiendas() {
    return "Lista de tiendas cargada";
}
// Es equivalente a:
// function obtenerTiendas() { return Promise.resolve("Lista de tiendas cargada"); 
}
  • Operador await:

    • Solo se puede usar dentro de funciones async. Detiene la ejecución de la función hasta que la promesa se resuelve, lo que hace que el código sea mucho más fácil de leer que las antiguas cadenas de .then(). Se detiene la ejecución solo en esta funcion el resto de código continúa

    🔹La forma antigua (Promesas con .then())

    
fetch('https://api.spagnolo.es/productos')
    .then(response => response.json())
    .then(data => console.log(data))
    .catch(error => console.error("Hubo un problema: ", error));

    🔹La forma moderna (async/await):

    
async function cargarProductos() {
    try {
        const response = await fetch('api/productos'); 
        // el sistema espera a que se tramite la peticion antes de seguir ejecutando este bloque, el resto de código sigue su ejecución
        if (!response.ok) {
            throw new Error(`Error inesperado: ${response.status}`);
          }
        const data = await response.json();   // Espera a que los datos se parseen
        console.log(data);  // el sistema espera a que se termine el parseo antes de seguir ejecutando este bloque
    } catch (error) {
        console.error("Error al obtener productos:", error);
    }
}
