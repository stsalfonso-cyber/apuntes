Funciones

Las funciones son bloques de código reutilizables que realizan una tarea específica.

  • Funciones declaradas

    Son las más comunes. Se definen con la palabra clave function seguida de un nombre. Tienen la ventaja de sufrir hoisting (elevación), lo que significa que pueden ser llamadas antes de ser declaradas en el código

    
function saludar(nombre) {
  return `Hola, ${nombre}!`;
}

  • Llamar a una funcion

    Para ejecutar una función, simplemente escribimos su nombre seguido de paréntesis. Si la función requiere argumentos, los colocamos dentro de los paréntesis. si la función devuelve algun valor se le asigna a una variable o dentro de una expresión

    
console.log(saludar("Alfonso")); // Imprime "Hola, Alfonso!"

  • Funciones expresadas

    Estas funciones se crean dentro de una expresión, normalmente asignándolas a una variable o constante. No tienen hoisting, por lo que debes declararlas antes de poder usarlas

    
const saludar = function(nombre) {
  return `Hola, ${nombre}!`;
};

  • Funciones flecha

    Introducidas en ES6, ofrecen una sintaxis más corta. Son muy útiles para callbacks (pasadas como argumentos) y no tienen su propio contexto this, heredándolo del ámbito donde fueron creadas

    
const saludar = (nombre) => {
  return `Hola, ${nombre}!`;
};

    Retorno implícito: Si la función tiene una sola expresión, no hay que poner llaves, ni parentesis en parámetros, ni usar return, se devuelve automáticamente el resultado de esa expresión

    
const duplicar = x => x * 2; 
console.log(duplicar(5)); // 10

    O sólo para ejecutar una acción sin devolver un valor

    
const saludar = nombre => console.log("Hola " + nombre);
saludar("Eva"); // Imprime "Hola Eva"

  • Funciones anónimas

    Son aquellas que no tienen un identificador o nombre. Se utilizan comúnmente como asignación a una variable, como argumentos de otras funciones (llamadas callbacks) o dentro de expresiones y funciones flecha

    Asignadas a una variable: 
    
const calcularArea = function(base, altura) {
  return base * altura;
};
    Como callback: (como método de setTimeout, map o filter) 
    
setTimeout(function() {
  console.log("¡Ejecutado después de 1 segundo!");
}, 1000);
    Función anónima y funcion flecha a la vez 
    
  // Función flecha principal
const obtenerCuadrados = (numeros) => {
    // Función anónima (arrow function) pasada como callback dentro del .map
  return numeros.map((numero) => {
    return numero * numero;
  });
};
console.log(obtenerCuadrados([2, 4, 6])); 
// Resultado: [4, 16, 36]
    Funciones Inmediatas (IIFE) (Inmediatly Invoked Function Expressions)

    Se definen y se ejecutan inmediatamente después de ser creadas. Se envuelven entre paréntesis para que JavaScript las trate como una expresión y no como una declaración

    
(function() {
  console.log("Esta función se ejecuta automáticamente al cargar.");
})();

    Estas funciones son útiles para crear un ámbito local y evitar la contaminación del espacio de nombres global, lo que ayuda a prevenir conflictos entre variables y funciones en proyectos más grandes (es habitual utilizar nombles génericos como url, contador... en cualquier otro lugar del código).

  • Callbacks

    Una función retrollamada, también conocida como callback, es una función que se pasa como argumento a otra función y se ejecuta una vez que se completa una tarea determinada. Muy usado en operaciones asíncronas, como peticiones de APIs, temporizadores o lectura de archivos, y también en manejo de eventos

  • Recursividad

    Consiste en que una función se llame a sí misma.

    function cuentaAtras(numero) {
                 // Condición base: Si el número que recibe es menor de 0 entonces salimos de la función>
                 if (numero < 0) { return }
                 // Si el número era mayor o igual a 0, lo mostramos 
                 console.log(numero) 
                 // y llamamos a la función con el número anterior
                 cuentaAtras(numero - 1)
               }

  • Closure y funciones de orden superior

    Las FOS son funciones que manejan funciones, reciben una función como argumento o devuelven una función resultado.

    
function crearDescuento(porcentaje) {
    // Esta variable 'porcentaje' vive aquí dentro
    return function(precio) {
      // La función interna "recuerda" el porcentaje aunque la externa ya se haya ejecutado
      return precio - (precio * porcentaje / 100);
    };
}
const descuentoBlackFriday = crearDescuento(20); // Aquí el descuento tiene un 20% guardado
// no hará falta pasar el porcentaje en cada llamada, sólo el importe
console.log(descuentoBlackFriday(100)); // Resultado: 80
console.log(descuentoBlackFriday(500)); // Resultado: 400

    La función descuentoBlackFriday no recibe el 20 por ciento directamente en cada llamada; lo recuerda gracias al closure.

    El objetivo del closure es recordar el entorno en el que fue creada la función y guardar el valor de las variables en ese entorno.
