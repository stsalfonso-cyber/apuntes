Try...Catch

El bloque try...catch es una estructura de control que permite manejar errores de manera elegante en JavaScript. Se utiliza para ejecutar código que podría generar una excepción (error) y proporcionar una forma de responder a ese error sin que el programa se detenga abruptamente.

Estructura básica:

try {
  // 1. Código que intentamos ejecutar ("probar")
} catch (error) {
  // 2. Este bloque se ejecuta SOLAMENTE si el código del 'try' falla
} finally {
  // 3. Este bloque se ejecuta SIEMPRE, haya habido error o no
}

Los métodos mas comunes son:

  • error.message: Es una cadena de texto que contiene el mensaje de error o la descripción de lo que ha fallado. Es la propiedad más utilizada para mostrar información al usuario final.
  • error.name: Es una cadena que indica el tipo de error. Por ejemplo: "TypeError", "ReferenceError", "SyntaxError" o simplemente "Error".
  • error.stack: Contiene un rastro (stack trace) detallado. Es una cadena de texto que enumera las llamadas a funciones que llevaron al error, mostrando los nombres de los archivos y los números de línea. Es indispensable para la depuración (debugging).
