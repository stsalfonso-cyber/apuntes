3. Ramas

Rama / Branch

Las ramas permiten que cada desarrollador (o cada tarea) viva en su propia realidad paralela.

1. Conceptos básicos

  • main (o master): Es la rama principal. Es la "realidad oficial". Todo lo que está aquí debe estar probado y funcionando.
  • Rama (Branch): Es un clon de main en un momento determinado. Al crear una rama, creas un espacio donde puedes hacer cambios sin afectar a la realidad oficial.

2 Los comandos más comunes


# 1. Crear una rama y saltar a ella inmediatamente
git checkout -b nombre-de-la-tarea

# 2. Listar ramas para saber dónde estás
git branch

# 3. Volver a la rama principal (antes de hacer un merge o pull)
git checkout main

(Nota: En versiones modernas de Git, git switch -c nombre-de-la-tarea es el comando preferido, pero checkout es el clásico que verás en todos sitios).

3 El Flujo de Trabajo (El "Workflow" de clase)

  1. Sincronizas: git pull origin main (Traes el código oficial al día).
  2. Ramificas: git checkout -b logo-spagnolo-nuevo.
  3. Trabajas: Haces tus cambios en CSS/HTML y haces tus git add + git commit.
