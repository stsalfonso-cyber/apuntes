EventListener

El DOM no es estático; el sitio web debe reaccionar

La sintaxis básica es: elemento.addEventListener(evento, manejador, [opciones]);


// La forma correcta: addEventListener
const btn = document.querySelector('#btn-compra');

btn.addEventListener('click', (event) => {
    console.log('Botón clickeado');
    // Prevenir comportamiento por defecto (ej. evitar que un formulario se envíe antes de validar)
    event.preventDefault(); 
});

  console.log(evento.type);    // 'click'
  console.log(evento.target);  // El botón clickeado

  evento.stopPropagation();    // Detiene la propagación del evento
});

Formularios

Los formularios son la herramienta principal para que el usuario envíe información a tu aplicación. En JavaScript no "enviamos" el formulario a la vieja usanza (recargando la página), sino que interceptamos el envío para procesarlo con nuestra lógica.

1. El evento submit y preventDefault()

Cuando un usuario pulsa el botón de "Enviar", el navegador intenta enviar el formulario a un servidor y recargar la página. ¡Esto es lo que queremos evitar!


const formulario = document.querySelector('#miFormulario');

formulario.addEventListener('submit', (e) => {
    // ESTO ES OBLIGATORIO: Detiene el envío automático del navegador
    e.preventDefault(); 

    console.log("Formulario interceptado, ahora lo proceso yo");
});

2. Recuperar datos:

Método A: Por nombre de campo (individual) 

const nombre = formulario.querySelector('#nombre').value;
console.log(nombre);
Método B: La API FormData (Nivel Pro)

FormData recolecta automáticamente todos los campos del formulario que tengan el atributo name.


/* HTML necesario: <input name="email" /> */

formulario.addEventListener('submit', (e) => {
    e.preventDefault();

    const formData = new FormData(formulario);

    // Convertir FormData a un objeto JS plano para trabajar cómodamente
    const datos = Object.fromEntries(formData.entries());

    console.log(datos); // { email: "usuario@ejemplo.com", password: "123" }
});

if (datos.email === "") {

Consejos a niviel profesional

En lugar de poner un "escuchador" a cada botón, se lo pones al padre. Fundamental si tienes una lista de miles de productos.


// Ponemos el evento en el padre (contenedor) y preguntamos por el hijo
const lista = document.querySelector('#contenedor-productos');

lista.addEventListener('click', (e) => {
    if (e.target.matches('.btn-producto')) {
        console.log('Click en un producto específico');
    }
});

Si necesitas añadir 100 productos a la lista, no hagas 100 append. Eso obliga al navegador a repintar la página 100 veces. Crea un contenedor invisible (fragmento) y añádelo de golpe.


            const fragment = document.createDocumentFragment();

for (let i = 0; i < 100; i++) {
    const li = document.createElement('li');
    li.textContent = `Producto ${i}`;
    fragment.append(li); // Se guarda en memoria, no repinta la página
}

// Un solo repaint
document.querySelector('#lista').append(fragment);

Atributo data (Dataset).Muy usado en e-commerce. Permite guardar datos técnicos en el HTML para que JavaScript los lea luego (ej. el ID de un producto).


<!-- En tu HTML -->
<div class="producto" data-id="12345" data-categoria="polos"></div>

<script>
// En tu JS
const prod = document.querySelector('.producto');
console.log(prod.dataset.id);        // "12345"
console.log(prod.dataset.categoria); // "polos"
</script>
