3. Datos del cliente


LocalStorage

El localStorage es una API del navegador que te permite guardar datos en el dispositivo del usuario de forma persistente. A diferencia de las variables de JavaScript, que se borran al refrescar la página, lo que escribas en localStorage seguirá ahí incluso si el usuario cierra el navegador o apaga el ordenador.

Métodos fundamentales

Estos son los comandos que usarás el 99% de las veces:


// 1. Guardar datos (Siempre clave-valor)
localStorage.setItem('tema', 'oscuro');

// 2. Recuperar datos
const temaActual = localStorage.getItem('tema'); // Devuelve 'oscuro'

// 3. Eliminar un elemento específico
localStorage.removeItem('tema');

// 4. Limpiar todo el almacenamiento de Spagnolo
localStorage.clear();

La capacidad limitada y el QuotaExceededError

El localStorage no es un disco duro infinito. Tienes un límite (normalmente 5MB por dominio).

Si superas el limite de memoria el sistema lanza una excepción QuotaExceededError.

Best Practice: Siempre envuelve las escrituras en un try...catch.


    try {
    localStorage.setItem('datosMuyGrandes', JSON.stringify(objetoGigante));
} catch (e) {
    if (e.name === 'QuotaExceededError') {
        console.error("¡Espacio lleno! Limpia el carrito.");
    }
}

JSON.stringify/parse

La Regla de Oro: Solo guarda Strings

El localStorage es muy estricto: solo puede guardar cadenas de texto. Si intentas guardar un objeto o un array directamente, el navegador lo convertirá a [object Object] y perderás la información.

Para guardar objetos se usa Json

  • Para guardar: Usas JSON.stringify(objeto) para convertirlo a string antes de guardarlo en localStorage
  • Para leer: Usas JSON.parse(string) para convertirlo a objeto y trabajar en js

El riesgo del "null"

Si intentas hacer JSON.parse(localStorage.getItem('claveNoExistente')), el resultado es null. Si luego intentas acceder a una propiedad, la web explota. Solución profesional:


const carrito = JSON.parse(localStorage.getItem('carrito')) || []; 
// Si localStorage devuelve null, usamos un array vacío como red de seguridad.

Sincronizacion entre pestañas


window.addEventListener('storage', (event) => {
  console.log(`La clave ${event.key} cambió de ${event.oldValue} a ${event.newValue}`);
  // Aquí podrías actualizar el DOM de la otra pestaña automáticamente
});

El evento storage es una herramienta clave para la comunicación entre pestañas o ventanas del mismo origen (mismo protocolo, dominio y puerto).

Es un error común pensar que este evento salta cada vez que se hace un localStorage.setItem(). No es así.

El evento storage solo se dispara en otras pestañas o ventanas que tengan tu sitio web abierto, nunca en la pestaña donde se ejecutó la acción.


  • Origen: Debe ocurrir un cambio en localStorage (o sessionStorage en versiones antiguas - específicas) en una pestaña distinta a la actual.
  • Acción: El evento se dispara al ejecutar: setItem, removeItem o clear
  • Cambio real: Si haces setItem('clave', 'valor') pero 'valor' ya es el valor actual en el localStorage, no salta el evento (porque no hubo cambio real).
  • Ejecución: En la pestaña que realizó el cambio, el evento no se ejecuta. Solo verás el evento en las otras pestañas abiertas del mismo dominio.

    sessionStorage VS localStorage

    Diferencia: localStorage VS sessionStorage

    En DAW es muy importante saber elegir cuál usar según el caso de uso:

    Característica localStorage sessionStorage
    Persistencia Permanente (hasta que se borra). Solo mientras la pestaña esté abierta
    Alcance Todo el dominio. Solo la pestaña actual.
    Uso común Preferencias de usuario, carritos de compra, tokens de login. Datos temporales de formularios, navegación de un solo uso.
