5. Mejores Prácticas de Arquitectura y Flujo


Separación de responsabilidades (Lógica vs UI)

El error más común del estudiante es mezclar el cálculo con la visualización.
¿Por qué es malo? Si mañana decides cambiar el <div> por un <section> o si quieres aplicar el mismo cálculo en otra parte de tu web, tendrás que reescribir toda tu lógica.
El patrón correcto: Divide siempre en:

  1. Lógica (Funciones puras): Reciben datos y devuelven datos. No conocen el DOM.
  2. UI (Interacción/DOM): Escuchan al usuario, llaman a la lógica y pintan el resultado.


// ❌ MAL: Mezclando todo
function calcularIVA() {   // función que lo mezcla todo
  const precio = document.querySelector('#precio').value;
  document.querySelector('#total').innerText = precio * 1.21;
}

// ✅ BIEN: Separado
const calcularIvaLogica = (precio) => precio * 1.21; // función que hace la lógica

const renderizarIva = () => { // función que toca el DOM
  const precioInput = document.querySelector('#precio').value;
  const resultado = calcularIvaLogica(precioInput);
  document.querySelector('#total').textContent = resultado;
};

Validación y Sanitización (Seguridad ante todo)

Nunca confíes en lo que viene del DOM. Los atacantes pueden inyectar código malicioso a través de formularios (ataque XSS).

  • Validación: ¿El formato es correcto? (Ej: ¿Es un email? ¿Está vacío?).
  • Sanitización (Escapar input): Nunca uses innerHTML si vas a mostrar contenido que escribió el usuario. Un usuario podría escribir <img src=x onerror=alert('hackeo')> y si haces div.innerHTML = input.value, ejecutas código malicioso.

Regla de oro:Para mostrar texto del usuario, siempre usa textContent, nunca innerHTML.


// ❌ PELIGROSO
const comentario = document.querySelector('#inputUser').value;
contenedor.innerHTML = comentario; // Si escribe etiquetas HTML, serán interpretadas

// ✅ SEGURO
const comentario = document.querySelector('#inputUser').value;
contenedor.textContent = comentario; // Lo trata como texto plano, inofensivo

Estructura y Orden de Archivos

En DAW no tendrás un solo archivo de 2000 líneas. Organiza tu proyecto desde el minuto uno para no volverte loco:


  1. index.html: Solo estructura y carga de scripts.
  2. style.css: Todo lo visual.
  3. main.js: El "Director de Orquesta". Aquí añades los EventListener y llamas a los demás módulos.
  4. api.js: Aquí van todos tus fetch.
  5. dom.js: Aquí van las funciones que solo tocan el DOM (renderizarLista, actualizarPrecio).
  6. utils.js: Funciones puras (cálculos matemáticos, validaciones, transformaciones).
