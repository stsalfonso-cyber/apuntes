4. Debugging & Professional Workflow


"El 90% de programar es resolver problemas. El debugger es tu linterna en la oscuridad."

Más allá del console.log

Otros métodos a parte del console.log...

  • console.table(arrayDeObjetos) Si recibes una lista de productos desde un fetch, no hagas un log gigante. table() te dibuja una tabla perfecta en consola donde puedes ordenar las columnas. Es obligatorio para trabajar con arrays de objetos.
  • console.dir(elementoDOM) Si haces un log de un div, te mostrará el HTML. Si haces dir, te mostrará todas sus propiedades de objeto (clases, eventos, estilo). Úsalo cuando quieras ver qué propiedades tiene un elemento que acabas de seleccionar.

Pestaña Network: El monitor de salud de tu API

Si un fetch falla, no pierdas tiempo en tu código antes de mirar aquí. Esta pestaña te dice exactamente qué está pasando en la "autopista" de datos:

  • Status Code: ¿Viene un 200 (OK)? ¿Un 404 (No encontrado)? ¿Un 500 (El servidor de Spagnolo ha petado)?
  • Preview / Response: Aquí puedes ver exactamente qué te ha devuelto el servidor (el JSON "crudo"). Útil para saber si el error está en tu código JS o si el servidor te está devolviendo algo que no esperabas.
  • Headers: vital para ver si estás enviando (o recibiendo) la autenticación correcta (ej: Authorization: Bearer...).

Pestaña Sources: Breakpoints y paso a paso

Aquí es donde controlas la ejecución del código como si fuera un video.


  1. Breakpoints (Puntos de interrupción): Haz clic en el número de cualquier línea en la pestaña Sources. La ejecución se detendrá ahí antes de ejecutar esa línea.
  2. Botones de control (La barra superior de Debug):
    • ◻ Step Over (la flecha saltando una línea): Ejecuta la siguiente línea. Si hay una función, la ejecuta de golpe sin entrar en ella.
    • ◻ Step Into (la flecha entrando hacia abajo): Si la línea tiene una función, entra dentro para que veas qué pasa en su interior.
  3. Scope (Ámbito): Mientras el código está detenido en un breakpoint, mira el panel lateral "Scope". Te dirá exactamente qué valor tiene cada variable en ese milisegundo. Es la mejor forma de cazar errores de variables undefined.

La palabra clave debugger;

Es tu "botón de pausa manual".


  async function procesarPago() {
    const total = 50;
    debugger; // <--- El navegador se detendrá aquí automáticamente al abrir la consola.
    // ...resto del código
}
  • Ventaja: No necesitas abrir DevTools y buscar el archivo; el navegador se pone en modo "Stop" tú solo. Es ideal cuando estás trabajando en un archivo gigante y quieres ir directo al kit de la cuestión.
