[3] Funciones y Modelo de Objetos (POO)

Funciones: Paso a paso

Argumentos: Por Valor vs. Por Referencia

  • Paso por valor (Por defecto): Si pasas una variable a una función, PHP crea una copia. Los cambios que hagas dentro de la función no afectan a la variable original.

              
    function incrementar($n) {
        $n++; 
    }
    $valor = 10;
    incrementar($valor); 
    echo $valor; // Sigue siendo 10

  • Paso por referencia (Con el símbolo &): Si quieres que la función modifique la variable original, añades un & antes del argumento.

            
  function incrementarReal(&$n) { 
    $n++; 
  } 
  $valor = 10; 
  incrementarReal($valor); 
  echo $valor; // ¡Ahora es 11!

Funciones Anónimas y "Closures"

Igual que en JS, puedes guardar funciones en variables y pasarlas como argumentos. Sin embargo, en PHP, si quieres que una función anónima use una variable que está fuera de su alcance, tienes que usar use.

    
$factor = 2;
// El 'use' es obligatorio para traer la variable de fuera
$multiplicar = function($n) use ($factor) {
    return $n * $factor;
};

echo $multiplicar(5); // 10

Recursividad

La recursividad en PHP funciona igual que en JS (una función que se llama a sí misma)

    
function factorial($n) {
    if ($n <?= 1) return 1;
    return $n * factorial($n - 1);
}

Nivel PRO

  • La importancia de los tipos. Aunque no es obligatorio, acostúmbrate siempre a tipar los argumentos que recibe y devuelve una función:

    function calcular(int $a, int $b): int { ... }.

    Esto evita que alguien pase un string al hacer un cálculo matemático.

  • Funciones como ciudadanos de primera clase: No tengas miedo de crear cientos de funciones pequeñas. En PHP es mucho mejor tener 10 funciones de 5 líneas cada una, que una función de 50 líneas.
  • El alcance (Scope): Recuerda que en PHP, las funciones son "islas". Una función no puede ver las variables definidas fuera a menos que las pases como argumento o uses la palabra clave global (esta última, evítala a toda costa, es mala práctica)


Programación Orientada a Objetos (POO)

La POO consiste en organizar tu código basándote en "entidades". En lugar de ver una web de Spagnolo como un montón de líneas de código, la vemos como un conjunto de objetos que interactúan: Usuario, Producto, Carrito, Pedido.


La esencia: ¿Qué es una Clase y qué es un Objeto?

1. La Clase: El "Plano" (Blueprint)

Una clase no es el producto. Una clase es la instrucción, el diseño, el plano.
Si tú escribes una clase Camisa, en ese momento no tienes ninguna camisa física en el almacén. Solo tienes un papel donde dice: "Una camisa debe tener una talla, un color, un precio y una función para calcular el IVA"

◼ La clase no ocupa memoria (no "existe" realmente en el sistema), es solo la definición.

◼ Regla de oro en POO. Un archivo PHP sólo debe contener una clase. Esto facilita: legitibidad, depuración y la colaboración con el resto de compañeros de programación.


2. Sintaxis Real en PHP"

A diferencia de JS, donde usabas this. y puntos, en PHP usamos $this-> (flecha). Esta es la diferencia más importante que debes tener en cuenta.

En function_contruct() lo que se hace es asignar el valor que viene de fuera a las variables internas.

    
class Producto {
    // 1. Propiedades (Lo que el objeto "tiene")
    public string $nombre;
    public float $precio;

    // 2. Constructor (Lo primero que se ejecuta al crear el objeto)
    //    Aquí se le asigna los valores que vienen de fuera
    public function __construct(string $nombre, float $precio) {
        $this->nombre = $nombre;
        $this->precio = $precio;
    }

    // 3. Método (Lo que el objeto "hace")
    public function aplicarDescuento(float $porc): void {
        $this->precio -= ($this->precio * $porc / 100);
    }
}

// Creamos un objeto (Instanciación)
$camisa = new Producto("Camisa Oxford", 50.00);
$camisa->aplicarDescuento(10); // Usamos la flecha "->"
echo $camisa->precio; // Imprime: 45

3. El Objeto: La "Instancia"

Cuando tú decides que necesitas una camisa real en tu sistema, haces lo que en programación llamamos instanciar. Es decir, sacas una copia física basada en el plano.

  • $camisa1 = new Camisa("Azul", "M", 50);
  • $camisa2 = new Camisa("Roja", "L", 55);

Aquí, $camisa1 y $camisa2 son objetos. Son entidades independientes que viven en la memoria de tu servidor. Tienen sus propios datos, pero comparten "la misma lógica" (la que definió la clase).

Este es el objetivo de la POO, agrupar estos datos (propiedades) y acciones propias (métodos) y relacionarlos con una entidad (objeto) y acceder desde ese objeto a sus propiedades y métodos facilmente


4. La Visibilidad (encapsulamiento)

En JS, todo es accesible. En PHP (y en Java), tú decides quién puede ver o tocar tus datos. Esto es obligatorio en DAW:

  • public: Todo el mundo puede acceder a la propiedad desde fuera con $camisa->precio.
  • private: Solo la misma clase puede ver $this->precio. Nadie de fuera puede tocarlo (ideal para proteger datos sensibles).
  • protected: Similar a privado, pero permite que las clases hijas (herencia) lo vean.

5. Reglas técnicas para POO en PHP

  • La Flecha ->: En PHP, el punto . es para concatenar strings. Por eso, para acceder a propiedades o métodos de un objeto, usamos la flecha ->.
  • $this: Representa a la instancia actual. Siempre que quieras tocar una propiedad o método de la clase, debes usar $this->.
  • Visibilidad: Acostúmbrate a marcar todo como private por defecto y solo hacerlo public si es estrictamente necesario. Esto se llama Encapsulamiento y es la base de la seguridad en Java y PHP.
  • Constructor Property Promotion: La declaracion de variables y asignacion de valores se hacen al mismo tiempo dentro de la function_construct() para ahorrar lineas.

Namespaces: La organización jerárquica del código

Los Namespaces actúan como carpetas virtuales para tu código. Permiten que aunque dos clases se llamen igual, vivan en "espacios" distintos y puedan coexistir.

1. La sintaxis es simple

Dentro de tu archivo PHP, lo primero (incluso antes que cualquier código) debe ser declarar a qué "espacio" pertenece ese archivo.

    
    // En el archivo Usuario.php
namespace Spagnolo\Models;

class Usuario {
    // ...
}

2. ¿Cómo los usamos? (La cláusula use)

No querrás escribir siempre el nombre completo (new Spagnolo\Models\Usuario()). Para eso existe use, que es exactamente lo mismo que el import de JavaScript.

    
namespace Spagnolo\Controllers;

// Importamos la clase (le decimos a PHP: "Cuando diga Usuario, mira en este Namespace")
use Spagnolo\Models\Usuario;

class LoginController {
    public function index() {
        $user = new Usuario(); // PHP sabe que es el de Spagnolo\Models
    }
}

3. El estándar DAW: PSR-4

"Si tu namespace es Spagnolo\Models\Usuario, tu archivo debe estar físicamente en la carpeta /Spagnolo/Models/Usuario.php."
